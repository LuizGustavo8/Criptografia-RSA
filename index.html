Fundamentos Matemáticos

RSA (Rivest, Shamir, Adleman, 1977) — sistema de chave pública baseado na dificuldade de fatorar n = p \cdot q

Geração de Chaves

  1. Escolher dois primos distintos p e q
  2. Calcular n = p \cdot q e \varphi(n) = (p-1)(q-1)
  3. Selecionar e coprimo a \varphi(n)
  4. Calcular d \equiv e^{-1} \pmod{\varphi(n)} via Euclides Estendido

Cifração e Decifração

Chaves
Pública:\ (n, e) \qquad Privada:\ (n, d)
Cifrar / Decifrar
c \equiv m^e \pmod{n} \qquad m \equiv c^d \pmod{n}
Prova (Euler-Fermat)
(m^e)^d = m^{1+k\varphi(n)} \equiv m \pmod{n}

CRT O Teorema Chinês do Resto acelera a decifração ~4×: calcula separadamente mod p e mod q, depois recombina.

Assinaturas Digitais e Variantes

Assinatura Digital
Assinar:\ s \equiv h(m)^d \pmod{n} \qquad Verificar:\ s^e \pmod{n} \stackrel{?}{=} h(m)

RSA-OAEP

CIFRAÇÃO

Padding probabilístico (PKCS#1 v2). Adiciona aleatoriedade antes de cifrar → previne ataques determinísticos.

RSA-PSS

ASSINATURA

Assinatura com padding probabilístico e prova formal de segurança redutível ao problema RSA.

Multiprime

PERFORMANCE

n = p_1 \cdot p_2 \cdots p_k (3+ primos). Decifração mais rápida via CRT, mas primos menores.

RSA + CRT

PERFORMANCE

m_p = c^{d \bmod p-1} \bmod p, m_q = c^{d \bmod q-1} \bmod q → ~4× mais rápido.

Esquemas Híbridos

RSA é ~1000× mais lento que AES. Na prática:

  1. Gerar chave AES aleatória (chave de sessão)
  2. Cifrar mensagem com AES (rápido)
  3. Cifrar apenas a chave AES com RSA

Combina velocidade do AES com segurança do RSA.

Square-and-Multiply

Exponenciação modular eficiente:

Converte expoente em binário. Para cada bit: quadra o acumulador. Se bit = 1: multiplica pela base.

O(e) \longrightarrow O(\log e)\ \text{multiplicações}

Ataque 1 — Divisão por Tentativa

O(√n) — exponencial

Testa todos os inteiros de 2 até \sqrt{n} como divisores. O mais intuitivo e antigo.

Python def divisao_tentativa(n): for d in range(2, isqrt(n) + 1): if n % d == 0: return d # fator! return n # n é primo
Exemplo: Fatorar 91
91 ÷ 2? Não (ímpar)
91 ÷ 3? Não (9+1=10)
91 ÷ 5? Não
91 ÷ 7? SIM → 91 = 7 × 13 ✓
Parou antes de √91 ≈ 9.5

Limite Prático

Para n com 100 dígitos:

\sqrt{n} \approx 10^{50}\ \text{divisões}

A 10^{10} div/s → 10^{40} segundos.
Idade do universo: \sim 4 \times 10^{17} s.
Ou seja: ~10²² vezes a idade do universo!

Útil como primeiro filtro: se n tem fator primo < 10⁶, encontra rápido.

Ataque 2 — Método ρ de Pollard

O(n1/4)

Gera sequência pseudoaleatória x_{i+1} = x_i^2 + 1 \bmod n. Pelo paradoxo do aniversário, após \sim\sqrt{p} iterações, dois valores colidem mod p. Detecta com o algoritmo da lebre e tartaruga (Floyd).

Python def rho_pollard(n): x = y = 2 d = 1 f = lambda x: (x*x + 1) % n while d == 1: x = f(x) # tartaruga: 1 passo y = f(f(y)) # lebre: 2 passos d = gcd(abs(x-y), n) if d != n: return d # fator!
Exemplo: n = 8051
f(x) = x² + 1 mod 8051
x₀ = 2
x₁ = f(2) = 5
x₂ = f(5) = 26
x₃ = f(26) = 677
...
Iteração 8:
  tartaruga = 2871
  lebre = 1671
  gcd(|2871−1671|, 8051)
  = gcd(1200, 8051) = 97
8051 = 83 × 97 ✓

Ataque 3 — Método p−1 de Pollard

Depende de B

Se (p-1) é B-smooth (todos os fatores primos ≤ B), pelo Pequeno Teorema de Fermat: a^{p-1} \equiv 1 \pmod{p}.

Ideia Central
M = \text{lcm}(1, 2, \ldots, B) \quad\Longrightarrow\quad p \mid \gcd(a^M - 1,\ n)
Python def p_menos_1(n, B): a = 2 for q in primos_ate(B): e = floor(log(B) / log(q)) a = pow(a, q**e, n) d = gcd(a - 1, n) if 1 < d < n: return d # fator!
Exemplo: n = 5917, B = 5
M = lcm(1,2,3,4,5) = 60
a = 2
Passo 1: a = 2⁴ mod 5917 = 16   (q=2)
Passo 2: a = 16³ mod 5917 = 4096 (q=3)
Passo 3: a = 4096⁵ mod 5917 = 3309 (q=5)
gcd(3309 − 1, 5917) = gcd(3308, 5917)
= 61 → fator!
5917 = 61 × 97 ✓
61−1 = 60 = 2²×3×5 → é 5-smooth!

Defesa Usar primos seguros (p = 2q+1, q primo), garantindo que p−1 tem fator primo grande.

Ataque 4 — Crivo Quadrático (QS)

Sub-exponencial Ln[1/2]
Objetivo
x^2 \equiv y^2 \pmod{n},\quad x \not\equiv \pm y \pmod{n} \quad\Longrightarrow\quad \gcd(x-y,\, n)\ \text{é fator de } n

Passos do Algoritmo

1 Escolher base de fatores B = {2, 3, 5, 7, 11, …}
2 Para x próximos de \sqrt{n}, calcular Q(x) = x^2 - n
3 Manter Q(x) que fatoram sobre B (são "B-smooth")
4 Representar fatorações como vetores mod 2
5 Eliminação gaussiana sobre GF(2) → combinação com expoentes pares
6 \gcd(x - y,\, n) → fator!
Exemplo Simplificado: n = 15
B = {2, 3}, √15 ≈ 3.87
Q(4) = 16−15 = 1 → {}
Q(5) = 25−15 = 10 = 2·5 → não B-smooth
Q(7) = 49−15 = 34 → não
Q(8) = 64−15 = 49 → não

(Em n real, centenas de relações
são combinadas via álgebra linear)

Eficiente até ~100 dígitos. Foi o mais rápido antes do GNFS (anos 1990).

Ataque 5 — GNFS

Ln[1/3, 1.923] — o mais rápido clássico

O algoritmo de fatoração mais eficiente para computação clássica. Generaliza o Crivo Quadrático usando corpos de números algébricos.

Etapas

1
Seleção de Polinômio
Encontrar f(x) grau d, m \in \mathbb{Z} tal que f(m) \equiv 0 \pmod{n}
2
Peneiramento (Sieving)
Buscar pares (a,b) smooth no lado racional E algébrico
3
Álgebra Linear
Sistema sobre GF(2) → encontrar dependência
4
Raiz Quadrada
Calcular raiz em corpo de números → x^2 \equiv y^2 \pmod{n}

Ataque 6 — Algoritmo de Shor

O((log n)³) — POLINOMIAL!

Usa computador quântico para encontrar o período r de f(x) = a^x \bmod n via Transformada Quântica de Fourier (QFT). Com r, a fatoração é direta.

Pseudocódigo def shor(n): a = random(2, n-1) if gcd(a, n) > 1: return gcd(a, n) # sorte! r = quantum_period(a, n) # QFT if r % 2 == 0 and pow(a, r//2, n) != n-1: p = gcd(a**(r//2) - 1, n) q = gcd(a**(r//2) + 1, n) return p, q

Status Atual

Hoje

IBM fatorou 15 e 21 com qubits reais. Números RSA reais: ainda impossível.

Necessário

RSA-2048 → ~4.000 qubits lógicos (~20 milhões de qubits físicos com EC)

Previsão

Entre 2035 e 2050. Estratégia: criptografia híbrida (RSA + pós-quântico)

Ataques por Falhas de Implementação

Estes ataques NÃO quebram a matemática do RSA. Exploram erros na forma como ele é usado.

Ataque de Wiener

d pequeno

Se d < n^{1/4}/3, frações contínuas de e/n revelam d. Convergentes k_1/d_1, k_2/d_2 \ldots — testar cada d_i como candidato.

Defesa: nunca usar d pequeno.

Ataque de Hastad

broadcast

Mesma msg m cifrada com e=3 para 3 destinatários.

CRT \Rightarrow m^3 \bmod (n_1 \cdot n_2 \cdot n_3) \Rightarrow \sqrt[3]{m^3} = m

Defesa: SEMPRE usar OAEP.

Bleichenbacher

PKCS#1 v1.5

Ataque CCA2 adaptativo. Modifica texto cifrado → servidor diz "padding válido/inválido" (1 bit). ~10⁶ consultas → mensagem inteira recuperada!

Defesa: migrar para RSA-OAEP.

Canal Lateral

físico

Timing: mede tempo de decifração → infere bits de d.
Power: consumo de energia revela operações.
Fault: erro no CRT → \gcd(m'^e - c,\, n) = q

Defesa: RSA blinding, tempo constante, verificação pós-decifração.

Análise Crítica e Contextualização

Lição da História

Cada geração de ataques fortaleceu o padrão:
PKCS#1 v1.5 → Bleichenbacher → OAEP
d pequeno → Wiener → regras de geração
tempo variável → ataque lateral → padding probabilístico

RSA vs. ECC

NívelRSAECC
128 bits3072b256b

ECC é mais eficiente. TLS moderno prefere ECDHE.

Ameaça Quântica

Algoritmo de Shor → fatoração polinomial. Ainda não viável, mas questão de tempo. Estratégia: criptografia híbrida (RSA + pós-quântico) como transição.

Pós-Quântico (NIST 2024)

ML-KEM (CRYSTALS-Kyber): troca de chaves
ML-DSA (CRYSTALS-Dilithium): assinaturas
Baseados em problemas de reticulados, resistentes a computadores quânticos.

RSA continua seguro com chaves ≥ 2048 bits e implementação correta, mas a migração pós-quântica já começou.

Conclusão

Elegância Matemática

RSA articula aritmética modular, Euler-Fermat, Euclides Estendido e CRT em uma aplicação real e crítica.

Sofisticação dos Ataques

Da divisão por tentativa ao GNFS e Shor, cada algoritmo explora uma propriedade matemática diferente.

Implementação Importa

Wiener, Bleichenbacher e canal lateral mostram que a segurança depende de como o RSA é usado.

Futuro Pós-Quântico

ECC como alternativa eficiente hoje. CRYSTALS-Kyber/Dilithium como destino inevitável.

"A segurança do RSA não reside apenas na matemática — reside na disciplina de quem o implementa."
